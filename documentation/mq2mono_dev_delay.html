Using Delay
  • Delays are how we release execution manually back to EQ/C++
  • Delays are automatically called when using MQ.CMD or on the next MQ action if you have taken too long (40ms)
  • If you want EQ to update any value , like HP updates/etc you have to release to let the game do work.
  • Normally you don't have to worry about this over much as it is almost completely automatic. 


MQ.Delay(1000); //delay 1 sec

//wait 300 milliseconds until we are still, basically works like the macro delay
MQ.Delay(300, "${Bool[!${Me.Moving}]}");

//IsCasting is a method that returns a bool
MQ.Delay(500, IsCasting);
